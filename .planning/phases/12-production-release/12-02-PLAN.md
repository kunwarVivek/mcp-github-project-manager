<task type="auto">
  <name>Task 2: Create troubleshooting guide</name>
  <files>docs/TROUBLESHOOTING.md</files>
  <action>
Create docs/TROUBLESHOOTING.md with common issues and solutions.

Structure:
1. **Quick Diagnosis**
   - Health check command: `curl localhost:PORT/health` (if applicable)
   - Test connection: How to verify MCP server is running
   - Log locations and how to enable debug logging

2. **Common Issues**

   **Authentication Errors**
   - "Bad credentials" - Token invalid or expired
   - "Resource not accessible" - Missing token scopes
   - "Not Found" - Wrong owner/repo or no access

   **Connection Issues**
   - "ECONNREFUSED" - Server not running
   - "Timeout" - Network issues or slow response
   - Server doesn't start - Check build exists (`npm run build`)

   **MCP Client Issues**
   - Claude: Server not appearing - Check config path and JSON syntax
   - Cursor: Connection failed - Verify npx can run the package
   - VS Code: Extension not connecting - Check MCP extension settings

   **AI Feature Issues**
   - "AI provider not configured" - Set API key for provider
   - "Rate limited" - Reduce request frequency
   - Fallback mode active - AI features work with reduced quality

   **GitHub API Issues**
   - "Secondary rate limit" - Wait and retry
   - "GraphQL errors" - Check query parameters
   - Project not found - Verify project ID format

3. **Debug Mode**
   - How to enable: `LOG_LEVEL=debug`
   - What to look for in logs
   - How to report bugs with debug output

4. **Getting Help**
   - GitHub Issues link
   - Required info for bug reports
   - Community resources

Include copy-pasteable solutions for each issue.
  </action>
